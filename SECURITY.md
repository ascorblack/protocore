# Security Policy

## Supported versions

Only the latest release line is guaranteed to receive security fixes.

## Reporting a vulnerability

Please do not open public issues for undisclosed vulnerabilities.

Instead:

1. Open a private GitHub security advisory (preferred), or
2. Contact maintainers through a private channel available in the repository profile.

Include:

- affected version and environment;
- reproduction steps or proof of concept;
- impact assessment;
- suggested fix (if available).

## Response targets

- Initial acknowledgement: within 3 business days.
- Triage status update: within 7 business days.
- Fix timeline depends on severity and complexity.

I will coordinate disclosure and release notes after a fix is available.
